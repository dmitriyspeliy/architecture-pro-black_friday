# pymongo-api

## Описание

Проект представляет собой API-сервис на FastAPI, работающий с MongoDB.

В рамках проектной работы архитектура была доработана для повышения:
- производительности;
- отказоустойчивости;
- масштабируемости.

В итоговом решении реализованы:
- **шардирование MongoDB** на 2 шарда;
- **репликация каждого шарда** в виде Replica Set из 3 узлов;
- **Replica Set для config servers**;
- **маршрутизация запросов через mongos**;
- **кеширование запросов в Redis**.

---

## Итоговая архитектура

В проекте используются следующие сервисы:

### Приложение
- `pymongo_api` — API на FastAPI

### Кеш
- `redis` — кеширование запросов приложения

### MongoDB sharded cluster
- `mongos` — роутер запросов к шардированному кластеру
- `configSrv1`
- `configSrv2`
- `configSrv3`

### Шард 1
- `shard1-1`
- `shard1-2`
- `shard1-3`

### Шард 2
- `shard2-1`
- `shard2-2`
- `shard2-3`

## Что было изменено

По сравнению с исходной версией проекта были внесены следующие изменения:

1. Убрана одиночная MongoDB-нода.
2. Добавлен шардированный кластер MongoDB.
3. Созданы 2 шарда для распределения данных.
4. Для каждого шарда настроен Replica Set из 3 узлов.
5. Добавлен Replica Set из 3 config servers для хранения метаданных кластера.
6. Приложение переведено на подключение через `mongos`.
7. Добавлен Redis для кеширования запросов.
8. В коде приложения блокирующий `time.sleep(1)` заменён на `await asyncio.sleep(1)`.

## Запуск проекта

### 1. Собрать и запустить контейнеры

```bash
docker compose up -d --build
```

### 2. Инициализировать MongoDB кластер

```bash
bash scripts/mongo-init.sh
```

### 3. Проверка работы

Проверка API

Откройте в браузере:

http://localhost:8080

Или выполните:

curl http://localhost:8080

Ожидаемые признаки корректной работы:

mongo_topology_type = "Sharded"
mongo_is_mongos = true
cache_enabled = true

### Проверка кеша

Первый запрос будет медленнее, повторный — быстрее.

curl http://localhost:8080/helloDoc/users
curl http://localhost:8080/helloDoc/users
